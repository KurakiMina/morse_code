# モールス信号練習

提示された旧UIコードと参考画像をベースに、クラシックモードをボード型UIとして復元した版です。

## ファイル

- `index.html`
- `styles.css`
- `app.js`

## 実装内容

- モード選択
- クラシックモード
  - 参考画像に寄せた `MORSE CODE` ボード
  - ● = 短押し
  - ■ = 長押し
  - 通過済み・現在位置を塗りつぶし
  - 数字チップを下段に表示
- ゲームモード
  - 40秒制限
  - ランダム単語
  - 1文字ずつ判定
  - スコア、コンボ、ランキング5位保存

## GitHub Pages

リポジトリ直下に3ファイルを配置してください。

```text
index.html
styles.css
app.js
```


## 今回の復旧

- クラシックモードは固定
- 練習モードを復活
- ゲームモードUIを復旧
- ゲームモードに40秒タイマー、スコア、コンボ、結果画面、ランキングを再追加


## クラシックモード調整

- O/M/Q/G/Z 周辺を修正
- Q の重複表示を解消
- Z を正しいモールスコード `--..` 側に配置
- `----` 側の誤ったQ表示を非表示化


## 今回の追加修正

- クラシックモードは固定
- 練習モードとゲームモードの「押す」ボタン内に押下時間バーを追加
- 短押し/長押しの境界が視覚的に分かるように調整
- 練習モード・ゲームモードにSEを追加
- 音声ON/OFFボタンを追加


## クラシックモードSE追加

- クラシックモードのUI・座標・表示ロジックは固定
- クラシックモードにもSEを追加
- 短押し音・長押し音をモールス信号らしい同一ピッチのトーンに調整
- 短押しは短いトーン、長押しは約3倍の長いトーン


## 第2段階の実装

- 連続正解数による難易度上昇
- 長い単語の追加
- 数字入り単語の出題
- 称号表示
- 正解・単語クリア・ミス時の演出強化
- クラシックモードと練習モードで6入力目以降は1秒後に自動破棄
- クラシックモードのUI・座標・表示は固定


## ゲームモードUX改善

- 入力中の経路補正を廃止
- 誤った方向に入っても即ミス扱いにしない
- 正解コードと一致した時だけ自動正解
- ミス判定はスペース確定時、または長すぎる入力時のみ
- G（--.）など、途中まで同じ経路をたどる文字の入力阻害を改善


## ゲームモードUX再精査

- 正解コードと一致した瞬間に、確実に次の文字へ進むように判定処理を分離
- 正解処理を `acceptGameCharacter` に分離し、単語クリア処理も独立化
- 現在入力と目標コードを並べて表示
- 入力済み記号を ● / ■ のチップで表示
- 入力中・不一致・一致状態が視覚的に分かるように強調
- 確定ボタンの表記を「スペース」から「確定」に変更


## ゲームモード入力表示の不具合修正

- 正解直後に `game.input` が即時クリアされ、「現在の入力」が未入力に見える問題を修正
- 正解・ミス直前の入力を `直前の入力` として短時間保持
- `現在の入力` も直前入力を一時表示するように変更
- 入力済みチップも正解直後に短時間残るように変更


## ゲームモード入力UXの再修正・検証

- 「確定」ボタンを廃止し、「入力クリア」に変更
- ゲームモードに明示入力ボタン「短押し ●」「長押し ■」を追加
- 「押す」ボタンのpointer captureを追加し、pointerup取りこぼしを防止
- document側にもpointerup保険を追加
- 入力中は即座に「現在の入力」「入力済みチップ」に反映
- 正解時は直前入力として短時間保持
- G（■■●）とN（■●）の入力遷移をブラウザ上で検証


## ゲームモード不具合修正

- 入力中に `setGameMessage()` が未定義で停止していた不具合を修正
- これにより、入力後に `updateGame()` まで到達せず「現在の入力」が未入力のままになる問題を修正
- 確定ボタンは廃止し、入力クリアへ変更
- 明示入力用の「短押し ●」「長押し ■」ボタンを追加
- `G = ■■●`、`N = ■●` の状態遷移を簡易検証
